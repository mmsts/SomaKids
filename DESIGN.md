# DESIGN.md — SomaKids 儿童身体感受翻译器

## 1. Overview

SomaKids 是一款帮助 4–12 岁儿童识别、表达身体感受的工具，使用者是孩子与他们的家长。

**核心情绪关键词**:`calm`(平静) · `soft`(柔和) · `safe`(安全) · `expressive`(表达感)

**视觉气质**:在深邃的夜空底色上漂浮着柔和的光晕。像一本会发光的睡前故事书,而不是一份诊疗报告。儿童感觉被拥抱,家长感觉被信任。

**设计原则**:
- **去医疗化**:避免任何让人联想到医院、问诊、化验单的视觉元素(白底 + 红字 + 数据表格 = 禁区)
- **可呼吸的留白**:每个元素都有充足的喘息空间,不让孩子感到信息逼近
- **可触可摸的质感**:玻璃、光晕、渐变让界面像柔软的物体而非冰冷的屏幕
- **温度而非锐度**:使用圆润、模糊、慢动效,避免硬边、闪烁、强对比
- **诗意大于功能**:信息层级用色彩与光感引导,而非加粗与边框

**风格参考**:
- **Fly**:浮层、光感、渐变、未来感
- **Apple Health**:克制、信息密度控制、温柔的色彩节制
- **童书插画**:圆润比例、情感化色彩、有机形状

## 2. Colors

整体采用「深空夜幕 + 暖色情绪光」的双层色彩结构。深色背景提供安全感与沉浸感,柔和情绪色作为前景信号。

### 2.1 Background — 背景层

| Token            | Hex         | 使用场景                |
| ---------------- | ----------- | ----------------------- |
| `bg/base`        | `#0E1033`   | 全局最底层,深蓝紫夜空  |
| `bg/elevated`    | `#1A1B47`   | 次级背景,分区/抽屉     |
| `bg/raised`      | `#252862`   | 卡片之下的容器层        |
| `bg/overlay`     | `#0E1033` @ 90% | 模态遮罩            |

> 禁止使用纯黑 `#000000` 与纯白 `#FFFFFF` 作为大面积背景。纯色会破坏柔和氛围。

## 3. Typography

### 3.1 字体族

| 用途              | 中文                  | 英文 / 数字       | 备注                       |
| ----------------- | --------------------- | ----------------- | -------------------------- |
| Display / Headline| PingFang SC           | SF Pro Rounded    | 圆润字形,避免严肃         |
| Body              | PingFang SC / 思源黑体 | Inter / Nunito    | 字重适中                   |
| 数字 / 计时       | —                     | SF Pro Rounded    | 仅用于年龄、时间等         |
| 装饰 / 引语       | 霞鹜文楷 / 站酷快乐体 | Caveat            | 仅用于引导语、签名感场景   |

> 优先选用「圆润几何 sans-serif」。**禁止**使用衬线、宋体、仿宋等带笔锋字体作为主字体。

### 3.2 字号阶梯(基于 4pt 网格)

| Token            | 字号 / 行高 | 字重 | 用途                  |
| ---------------- | ----------- | ---- | --------------------- |
| `text/display`   | 36 / 44     | 700  | 引导页主标题          |
| `text/h1`        | 28 / 36     | 700  | 页面主标题            |
| `text/h2`        | 22 / 30     | 600  | 区块标题              |
| `text/h3`        | 18 / 26     | 600  | 卡片标题              |
| `text/body`      | 16 / 26     | 500  | 正文(儿童阅读基线)  |
| `text/bodyL`     | 18 / 28     | 500  | 儿童交互区主文字      |
| `text/caption`   | 14 / 20     | 500  | 辅助说明              |
| `text/label`     | 13 / 18     | 600  | 标签 / 按钮文字       |

### 3.3 字距与对齐

- 中文 `letter-spacing`:`0.02em`(轻微展开,更柔和)
- 英文标题 `letter-spacing`:`-0.01em`(紧凑现代)
- 默认左对齐;引导/欢迎页可居中
- 段落最大宽度 28em,避免儿童视觉跳行

### 3.4 文字使用约束

- 儿童交互区最小字号 **16px**,禁止 `< 14px`
- 单段文字 ≤ 2 行(儿童视图)
- 禁止全大写英文标题(除非 ≤ 3 字母)
- 不使用纯白 `#FFFFFF`,统一使用 `text/primary` `#F4F4FF`

## 4. Layout

### 4.1 网格

- 基础网格:**4pt**
- 内边距阶梯:`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 56 / 80`
- 容器最大宽度:移动端 100%(左右各 20px 安全边距),平板 720px

### 4.2 安全边距(Safe Zone)

| 层级              | 数值        | 备注              |
| ----------------- | ----------- | ----------------- |
| 屏幕水平边距      | 20px        | 不可贴边          |
| 顶部状态栏外延    | 12px        |                   |
| 卡片内边距        | 20–24px     | 儿童触摸友好      |
| 卡片之间间距      | 16px        |                   |
| 区块之间间距      | 32–40px     | 给信息留呼吸      |

### 4.3 触摸目标

- 最小可点击尺寸:**48 × 48 px**(儿童手指容错)
- 主操作按钮高度:**56 px**
- 图标按钮:**44 × 44 px**
- 相邻可点击元素之间间距 ≥ 8px

### 4.4 层级结构

界面遵循 **「天空 — 云层 — 卡片 — 内容」** 四层结构:
1. **天空(base)**:深空背景 + 极远光斑
2. **云层(atmosphere)**:大面积模糊渐变光(柔和发光,无边界)
3. **卡片(surface)**:玻璃质感容器,承载结构化信息
4. **内容(content)**:文字、图标、插画

### 4.5 滚动与节奏

- 垂直滚动为主,**禁止**横向滚动通栏切屏(眩晕风险)
- 每屏首屏必须有清晰主角元素(一个核心卡片 / 插画)
- 列表使用大间距(≥ 16px),不使用密集表格

## 5. Elevation & Depth

深色 UI 不使用传统投影,而是通过 **光晕 + 模糊 + 透明度** 营造层次。

### 5.1 层级

| Level | 作用                | 视觉手法                       |
| ----- | ------------------- | ------------------------------ |
| L0    | 底层背景            | `bg/base`                      |
| L1    | 区块容器            | `bg` + 5% 白色叠加             |
| L2    | 卡片                | 玻璃模糊 + 1px 顶部高光        |
| L3    | 浮层 / 弹窗         | 玻璃模糊 + 外部光晕            |
| L4    | 焦点元素 / 引导提示 | 渐变光晕 + 微动                |

### 5.2 玻璃材质(Glass / Frosted)

- 背景模糊:`backdrop-blur: 20–32px`
- 表面填充:`rgba(255, 255, 255, 0.06)` 至 `rgba(255, 255, 255, 0.12)`
- 顶部 1px 高光:`linear-gradient(180deg, rgba(255,255,255,0.20), transparent)` 仅顶部 1px
- 边框:`1px solid rgba(255, 255, 255, 0.08)`

### 5.3 光晕(Glow)

| Token           | 形态                                           |
| --------------- | ---------------------------------------------- |
| `glow/soft`     | `0 0 40px rgba(139, 127, 232, 0.25)`           |
| `glow/medium`   | `0 8px 60px rgba(139, 127, 232, 0.35)`         |
| `glow/strong`   | `0 0 80px rgba(164, 142, 255, 0.50)`           |
| `glow/emotion/*`| 跟随情绪色,半径 60–100px,透明度 30–50%       |

### 5.4 阴影(仅辅助)

深色界面慎用黑色投影(会显脏)。如需投影,使用:
- `shadow/sink`:`0 4px 24px rgba(14, 16, 51, 0.4)` —— 浮层向下沉底
- **禁止** `box-shadow: 0 2px 4px black` 这种小硬阴影

### 5.5 模糊度参考

- 远景光斑:`blur 80–120px`
- 卡片背景模糊:`blur 20–32px`
- 焦点元素背后辉光:`blur 40–60px`

## 6. Shapes

圆角是 SomaKids 的灵魂。所有元素遵循「无尖角」原则。

### 6.1 圆角阶梯

| Token         | 半径    | 用途                    |
| ------------- | ------- | ----------------------- |
| `radius/xs`   | 8px     | 标签 / 小芯片           |
| `radius/sm`   | 12px    | 输入框 / 小按钮         |
| `radius/md`   | 16px    | 标准按钮                |
| `radius/lg`   | 20px    | 卡片                    |
| `radius/xl`   | 28px    | 大卡片 / 弹窗           |
| `radius/2xl`  | 36px    | 引导页插画容器          |
| `radius/pill` | 9999px  | 胶囊按钮 / Tab          |
| `radius/full` | 50%     | 头像 / 情绪小球         |

### 6.2 比例规则

- 卡片圆角 ≥ 20px(儿童视觉友好)
- 按钮圆角 ≥ 16px
- 任何元素都不使用 `radius < 8px`,**禁止直角 0px**
- 圆角应当 **一致**:同层级元素圆角差异 ≤ 4px

### 6.3 形状语汇

- **主要图形**:圆、椭圆、圆角矩形、不规则有机斑块(blob)
- **辅助图形**:星点、月牙、波浪线
- **禁止**:尖锐三角、十字、警示标志、闪电符号

## 7. Components

### 7.1 Card — 卡片

**默认卡片**
- 背景:`rgba(255, 255, 255, 0.06)` + `backdrop-blur: 24px`
- 圆角:`radius/lg` (20px)
- 边框:`1px rgba(255, 255, 255, 0.08)`
- 顶部高光:1px 渐变高光
- 内边距:20–24px
- 可选:背后柔和光晕(`glow/soft`)

**情绪卡片(变体)**
- 背景:玻璃 + 情绪色光晕(`emo/*` 30% 透明度从底部弥漫)
- 顶部一个圆形情绪图标(直径 56px,带情绪色光晕)

> **禁止**:1px 灰色描边卡片、纯实色卡片、白底卡片

### 7.2 Button — 按钮

**Primary(主按钮)**
- 背景:`grad/aurora` (`#7B6BFF` → `#5DA8FF`, 135°)
- 文字:`#FFFFFF`,字重 600,字号 16px
- 高度:56px
- 圆角:`radius/md` (16px)
- 发光:`0 8px 32px rgba(123, 107, 255, 0.4)` + 内 1px 顶部高光
- Hover/Press:轻微缩放 0.97 + 光晕增强
- 禁用:透明度 40% + 关闭发光

**Secondary(次按钮)**
- 背景:`rgba(255, 255, 255, 0.08)` + 玻璃模糊
- 边框:`1px rgba(255, 255, 255, 0.16)`
- 文字:`text/primary`

**Ghost(透明按钮)**
- 仅文字,颜色 `brand/secondary`
- Hover 出现底色 8% 白色

**Pill / Chip**
- 圆角:`radius/pill`
- 高度:36px(成人) / 44px(儿童版)
- 内边距:左右 16px

### 7.3 Input — 输入框

- 背景:`rgba(255, 255, 255, 0.05)`
- 边框:`1px rgba(255, 255, 255, 0.12)`
- 圆角:`radius/sm` (12px)
- 高度:52px
- 内边距:左右 16px
- 文字:`text/primary` 16px(高对比)
- 占位符:`text/tertiary`
- Focus:边框过渡为 `brand/primary` + 外发光 `0 0 0 4px rgba(139, 127, 232, 0.2)`
- Error:边框 `state/error` + 下方 caption 提示

> **禁止**纯白底 + 黑边框的传统输入框。

### 7.4 Body Map — 身体地图

核心交互组件 —— 儿童点击身体部位描述感受。

- **形态**:柔和圆润的儿童身体轮廓插画(无解剖结构、无器官)
- **主体色**:`bg/raised` 渐变 + 半透明
- **可选区域**:被点击时显示对应情绪色光晕(`glow/emotion/*`)
- **周围漂浮 6–8 个情绪小球**(直径 32–40px,圆形,带情绪色 + 发光)
- 拖拽情绪小球到身体部位即可标记

### 7.5 Emotion Bubble — 情绪气泡

- 圆形,直径 40 / 56 / 72 px 三档
- 背景:对应 `emo/*` 渐变(中心亮,边缘半透明)
- 发光:`0 0 24px {emoColor at 50%}`
- 内部居中放置一个圆润 emoji 风格的图标或表情
- 可拖拽 / 可点击
- 选中态:放大 1.1 + 光晕增强

### 7.6 Voice Wave — 语音波形

录音组件 —— 让不识字的孩子也能用声音描述。

- 波形颜色:`grad/aurora` 渐变
- 高度变化:40–80px
- 背景:玻璃卡片
- 录音中:底部出现脉动光圈(`brand/primary` 50% 透明度,1.5s 呼吸)

### 7.7 Companion — 陪伴角色

整个 App 的「云朵小伙伴」—— 一个圆润的拟人云朵或小生物。

- **形态**:blob 不规则圆角,带柔和光晕
- **颜色**:`grad/dawn` 或随情绪变化
- **表情**:极简(两个圆点眼 + 一根曲线嘴)
- 用于引导、提示、空状态、加载等待
- **禁止**真实人物头像、医生卡通形象

### 7.8 Modal / Sheet — 弹层

- 从底部上滑(移动端 Sheet)
- 顶部圆角 `radius/xl` (28px),底部直角
- 顶部 4×40px 浅灰拖拽条(`rgba(255,255,255,0.2)`)
- 背景:玻璃模糊(`backdrop-blur: 32px`)+ `rgba(26, 27, 71, 0.85)`
- 遮罩:`rgba(14, 16, 51, 0.6)`

### 7.9 Tab Bar — 底部导航

- 容器:玻璃模糊 + 顶部 1px 高光
- 高度:72px(含安全区)
- 选中态:图标变为 `grad/aurora` 渐变 + 上方 6px 圆形光点
- 默认态:`text/tertiary` 单色
- 圆角图标,**禁止**线性细描边图标(用 filled 圆润款)

### 7.10 Tag / Chip

- 高度:32px
- 内边距:12px
- 圆角:`radius/pill`
- 背景:对应情绪色 20% 透明度
- 文字:对应情绪色实色(如 `emo/joy` 文字 + `emo/joy` @ 20% 背景)

### 7.11 Empty State — 空状态

- 中央放置 Companion 角色,表情温和
- 下方一句温柔的引导语(不超过 12 字)
- 一个柔和的 CTA 按钮
- **禁止**「数据为空」「暂无内容」此类冷漠文案

### 7.12 Loading

- **禁止**传统 spinner
- 使用脉动光圈:圆形,`brand/primary` 50%,1.2s 呼吸
- 或使用三个情绪小球的弹跳波动动画
- 时长 > 2s 时配合 Companion 出现一句安抚语

### 7.13 Animation 原则

- 默认时长:240–320ms(儿童视觉舒适)
- 缓动:`cubic-bezier(0.4, 0, 0.2, 1)` (柔顺) 或 `cubic-bezier(0.34, 1.56, 0.64, 1)` (轻微回弹)
- 进入:透明度 + 上浮 8px
- 选中:缩放 1.05 + 光晕增强
- **禁止**:硬切换、闪烁、震动、>500ms 的等待动画

## 8. Do's and Don'ts

### Do

- 使用深空背景 + 柔和光晕营造夜灯般的安全感
- 卡片始终是玻璃模糊质感,让信息「漂浮」而非「贴死」
- 让光晕成为主角 —— 比起阴影,更多使用发光来分层
- 圆角永远 ≥ 8px,关键元素 ≥ 16px
- 用情绪色板传递感受,而不是用色块强行装饰
- 保持充足留白,每个元素都能呼吸
- Companion 角色出现在引导/空状态/加载中,给孩子陪伴感
- 文字最小 14px,儿童交互文字 ≥ 16px
- 动效轻盈、缓慢、可预测
- 始终把儿童作为第一阅读者:用插画 + 颜色 + 声音组合表达
- 使用比喻(「肚子里有蝴蝶」「胸口暖暖的」)替代医疗术语

### Don't

- 不要使用纯黑 `#000000` 或纯白 `#FFFFFF` 大面积铺面
- 不要使用饱和原色(`#FF0000` / `#00FF00` / `#FFFF00`)
- 不要使用医疗符号(红十字、听诊器、化验单图标、药片图标)
- 不要用方表格、网格分割线、Excel 风格信息布局展示数据
- 不要使用尖锐图形(细长三角、警示惊叹号、闪电)
- 不要写「警告」「错误」「失败」「无效」这种冰冷词汇 —— 用「再试一次好吗?」「这次没听清呢」替代
- 不要使用直角 0px 与硬阴影 `0 2px 4px black`
- 不要让按钮贴边或多个按钮挤在一起(间距 ≥ 8px)
- 不要使用衬线、宋体、仿宋等带笔锋字体作为主字体
- 不要使用闪烁、抖动、>500ms 的硬切换动画
- 不要在儿童视图中出现长段文字(超过 2 行需重新设计)
- 不要在同一屏幕中出现 3 种以上情绪色 —— 信号要克制
- 不要使用真实照片或写实人物头像作为主视觉
